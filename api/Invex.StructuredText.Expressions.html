Namespace Invex.StructuredText.Expressions

Classes

AndExpression

Logical AND over Source. The platform formatter decides the rendered syntax (e.g. a && b or and(a, b)).

BooleanExpression

A boolean literal, rendered as true or false.

CastExpression<TTo>

Wraps an expression to reinterpret its result type at compile time. The formatter should treat this transparently — it simply delegates to the inner expression.

CoalesceExpression

Returns the first non-empty value of Source, rendered as coalesce(...).

ConcatExpression

Concatenates the rendered text of Values with no separator. Handled directly by TextExpressionFormatter, so it works on every platform.

ContainsExpression

Tests whether Source contains Pattern, rendered as contains(source, pattern).

EndsWithExpression

Tests whether Source ends with Pattern, rendered as endsWith(source, pattern).

EqualExpression

Equality comparison. The platform formatter decides the rendered syntax (e.g. left == right or eq(left, right)).

EvaluateExpression

Marks Expression for evaluation in the target platform's expression context.

FormatExpression

Replaces placeholders ({0}, {1}, …) in Source with Arguments, rendered as format(source, args...). Usually built via TextExpressions.Format($"...") string interpolation.

GreaterThanExpression

Greater-than comparison. The platform formatter decides the rendered syntax (e.g. left > right or gt(left, right)).

GreaterThanOrEqualToExpression

Greater-than-or-equal comparison. The platform formatter decides the rendered syntax (e.g. left >= right or ge(left, right)).

HashFilesExpression

Returns a hash of the files matched by the glob pattern(s) in Source, rendered as hashFiles(source). Commonly used for cache keys.

IndexAccessExpression

Accesses an element of Array by Index, rendered as array[index].

JoinExpression

Joins the elements of the array-valued Source with OptionalSeparator (comma when omitted), rendered as join(source, separator).

LessThanExpression

Less-than comparison. The platform formatter decides the rendered syntax (e.g. left < right or lt(left, right)).

LessThanOrEqualToExpression

Less-than-or-equal comparison. The platform formatter decides the rendered syntax (e.g. left <= right or le(left, right)).

NotEqualExpression

Inequality comparison. The platform formatter decides the rendered syntax (e.g. left != right or ne(left, right)).

NotExpression

Logical negation of Source. The platform formatter decides the rendered syntax (e.g. prefix ! or not(source)).

NullExpression

The null literal, rendered as empty text.

NumberExpression

A numeric literal, rendered using the invariant culture.

OrExpression

Logical OR over Source. The platform formatter decides the rendered syntax (e.g. a || b or or(a, b)).

PropertyAccessExpression

Accesses Property of Object, rendered as object.property.

RawExpression

Verbatim, uninterpreted text. The formatter writes Value exactly as-is. Use this for plain YAML values and context path references; use StringExpression for quoted string literals inside expressions.

StartsWithExpression

Tests whether Source starts with Pattern, rendered as startsWith(source, pattern).

StepOutcomeExpression

References the outcome of a step in the current job, rendered as steps.<step>.outcome. Compare against a StepOutcomeTypeExpression.

StepOutcomeTypeExpression

A step outcome literal for comparison with StepOutcomeExpression. The platform formatter renders the appropriate outcome string for the target platform.

StepOutputExpression

References an output of a step in the current job. The platform formatter renders the appropriate context path for the target platform.

StringExpression

A string literal inside an expression, rendered with quotes (e.g. 'value'). Use RawExpression for verbatim, unquoted text.

TargetOutcomeExpression

References the outcome (result/status) of another job or stage. The platform formatter renders the appropriate context path for the target platform. Compare against a TargetOutcomeTypeExpression.

TargetOutcomeTypeExpression

A job/stage outcome literal for comparison with TargetOutcomeExpression. The platform formatter renders the appropriate outcome string for the target platform.

TargetOutputExpression

References an output of another job or stage. The platform formatter renders the appropriate context path for the target platform.

TextExpression

The base type of the platform-agnostic expression tree. Expressions are immutable records composed via fluent methods and operators, then rendered to platform-specific text by a TextExpressionFormatter.

TextExpressionCollection

An ordered collection of TextExpression instances. Implicit conversions from string, string[], List<T>, and single expressions allow model properties to be assigned naturally, e.g. Run = ["dotnet build"] or Branches = ["main"].

TextExpressionFormatter

Base class for platform-specific expression formatters.

TextExpressionUtils

Utility helpers for working with sequences and trees of TextExpression.

TextExpressions

The entry point for creating TextExpression instances. Factory members (Raw, From, Format, Concat, True, False, Null, …) are attached to this class via extension members in WorkflowExpressionExtensions.

ToJsonExpression

Converts Source to its JSON representation, rendered as the platform's JSON serialization function.

WorkflowExpressionCollection<T>

A typed workflow expression collection that documents the expected element result type. The type parameter serves as a compile-time hint for what each expression resolves to.

WorkflowExpressionExtensions

Extension members that augment TextExpressions and TextExpression with factory helpers for building expression trees.

Structs

TextExpressionInterpolatedStringHandler

An interpolated string handler that creates a FormatExpression from an interpolated string. Literal parts are combined into a format string with placeholders, and TextExpression holes become arguments.

WorkflowExpression<T>

A typed workflow expression that documents the expected result type of the expression. The type parameter serves as a compile-time hint for what the expression resolves to.

Interfaces

ITextExpressionFormatter

Renders a TextExpression tree into platform-specific text.

ITextExpression<TResult>

Marker interface that documents the result type an expression is expected to resolve to. The type parameter is a compile-time hint only; it has no effect on formatting.
