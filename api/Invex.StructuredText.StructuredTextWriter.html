Namespace

An indentation-aware text writer for producing structured text such as YAML. Indentation is managed through disposable scopes returned by WriteSection(string?), so the nesting of the generating code mirrors the nesting of the generated output.

Version

A counter incremented by every Reset(int). Section scopes created before a reset compare against this value and become no-ops on dispose, so stale scopes cannot corrupt the indentation of content written after the reset.

WriteSection(string?)

public IDisposable WriteSection(string? text = null)

Parameters

text string

An optional header line (e.g. "steps:") written at the current indent level before the indent is increased. When null or empty, only the indent changes.

Returns

IDisposable

A scope that restores the previous indent level when disposed. Dispose it (typically via using) to close the section.

Remarks

If Reset(int) is called while a scope is still open, disposing that scope does nothing — the scope is tied to the Version it was created under.
